# NotebookLM 扩展功能更新 - v4.60

我完成了高级浮动搜索系统的重构，并修复了侧边栏的布局对齐问题。

## 已完成的主要更新

### 1. 高级浮动搜索系统与精细化交互 (v4.67/v4.66/v4.65/v4.64/v4.63)
- **视觉一致性 (v4.67)**：彻底统一了目录内文档与待分类文档的标题字体大小 (13px) 和颜色 (text-soft)，消除了由于 DOM 层级不同导致的视觉差异。
- **快捷交互 (v4.66)**：在新建目录的输入框中增加了对 `Esc` 键的监听，按下即可无损取消目录创建过程，避免误触烦恼。
- **类名统合 (v4.65)**：统一了文件夹内和待分类区域的文件标签类名均为 `.nb-ext-source-label`，使过滤选择器更精准、样式更统一。
- **极致过滤 (v4.64)**：解决了因为 CSS `!important` 导致的列表项无法隐藏的问题。通过在其 CSS 末尾定义 `.nb-ext-hidden` 实现了最高优先级的过滤显示。
- **过滤可靠性 (v4.63)**：修复了数据刷新后过滤失效的问题。
- **底部对齐 (v4.63)**：搜索面板优化至侧边栏底部，操作更便捷。
- **宽度随动 (v4.62)**：搜索面板宽度实时感知侧边栏缩放。
- **输入法友好 (v4.61)**：完美支持 IME (如拼音) 输入。
- **逻辑匹配**：深度支持 `空格 (OR)` 与 `+ (AND)` 逻辑。

### 2. 精准布局修复 (v4.54)
- **强制置顶**：通过 DOM 强制置顶“全选”容器，确保其始终位于列表最上方。
- **垂直流向**：修正了宿主容器的 Flex 布局方向，解决了元素意外并排的问题。

### 3. 主题与品牌回归 (v4.52/4.51)
- **亮暗适配**：基于 `body.dark-theme` 的全自动色彩变量切换。
- **经典图标**：回归 Google Red (#EA4335) 设计，并优化为 PNG 格式以提升浏览器兼容性。
