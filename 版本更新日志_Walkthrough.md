# NotebookLM 扩展功能更新 - v1.1.7

我优化了 Notebook 切换体验，移除了全页刷新，并大幅提升了 UI 视觉质量。

## 已完成的主要更新

### 1. 无刷新平滑切换 (v1.1.7)
- **SPA 深度适配**：重构了初始化逻辑，现在切换 Notebook 时，插件会自动重置内部状态并重新扫描数据，不再触发 `location.reload()`。这使得切换过程更顺滑，避免了白屏和等待感。
- **自动初始化修复**：解决了从首页进入 Notebook 时由于 URL 延迟可能导致扩展未加载的问题。

### 2. 彩色文件图标 (v1.1.7)
- **智能配色系统**：根据文件扩展名自动为图标分配颜色（如 PDF 为红色、表格为绿色、文档为蓝色），显著提升侧边栏的可读性和视觉丰富度。
- **暗色模式适配**：针对暗色主题优化了图标对比度，确保在任何背景下都能清晰辨识。

### 3. 自适应侧边栏高度
- **背景**：在内容较少时，原容器会固定占据整个侧边栏高度，遮挡底部的原生功能按钮（如“Add source”）。
- **优化**：将容器高度改为 `auto` 并配合 `max-height`。底部空白区域现在为透明，允许原生控制区显现，解决了“一大段空白”的视觉问题。

### 4. 修复文件夹一键反选（支持笔记）
- **核心修复**：修复了 `getSelectionState` 逻辑，现在能正确统计文件夹内的笔记（Notes）选中状态。
- **效果**：解决了文件夹显示为“部分选中”状态且点击无法清空的问题。现在点击文件夹图标可以完美实现“全选”与“全部清空”的切换。

## NotebookLM 扩展功能更新 - v1.1.2

## 已完成的主要更新

### 许可证验证系统修复 (v1.1.2)
- **解决 500 错误**：通过分析 API 日志，将许可证验证参数从 `product_permalink` 更改为 Gumroad 要求的 `product_id`，彻底解决了激活时的内部服务器错误。
- **统一支付域名**：将所有购买和验证链接的域名统一为 `gumroad.com`，规避了重定向导致的潜在验证失败，确保支付体验的连贯性。
- **增强日志诊断**：在验证流程中集成了详细的 HTTP 错误状态捕获，若未来发生异常，可快速定位问题。

# NotebookLM 扩展功能更新 - v4.60

我完成了高级浮动搜索系统的重构，并修复了侧边栏的布局对齐问题。

## 已完成的主要更新

### 1. 高级浮动搜索系统与精细化交互 (v4.73/v4.71/v4.70/v4.69/v4.67/v4.66/v4.65/v4.64/v4.63)
- **精简体验 (v4.73)**：去除了目前暂未实质性实现文件重排的基于文件名的排序按钮，使得工具栏更为简洁紧凑。
- **事件闭环 (v4.71)**：完善了搜索组件的生命周期。现在当用户点击 `x` 图标或使用 `Esc` 键关闭搜索面板时，内部的搜索状态和列表的过滤显示都会被瞬间清空和还原。
- **纯英语境 (v4.70)**：为了匹配 Google 的国际化应用生态，我们将所有的悬停提示、按钮文案和默认标题（如 `Unassigned (N)`）全部重构为地道纯正的英语。
- **智能全选 (v4.69)**：修复了搜索状态下全选逻辑的漏洞。现在在有搜索关键词过滤时，点击顶部的“全选”按钮只会选中**当前可见**的文档，真正实现了“过滤出我想要的，然后一键全选”的高效工作流。
- **视觉一致性 (v4.67)**：彻底统一了目录内文档与待分类文档的标题字体大小 (13px) 和颜色 (text-soft)，消除了由于 DOM 层级不同导致的视觉差异。
- **快捷交互 (v4.66)**：在新建目录的输入框中增加了对 `Esc` 键的监听，按下即可无损取消目录创建过程，避免误触烦恼。
- **类名统合 (v4.65)**：统一了文件夹内和待分类区域的文件标签类名均为 `.nb-ext-source-label`，使过滤选择器更精准、样式更统一。
- **极致过滤 (v4.64)**：解决了因为 CSS `!important` 导致的列表项无法隐藏的问题。通过在其 CSS 末尾定义 `.nb-ext-hidden` 实现了最高优先级的过滤显示。
- **过滤可靠性 (v4.63)**：修复了数据刷新后过滤失效的问题。
- **底部对齐 (v4.63)**：搜索面板优化至侧边栏底部，操作更便捷。
- **宽度随动 (v4.62)**：搜索面板宽度实时感知侧边栏缩放。
- **输入法友好 (v4.61)**：完美支持 IME (如拼音) 输入。
- **逻辑匹配**：深度支持 `空格 (OR)` 与 `+ (AND)` 逻辑。

### 2. 精准布局修复 (v4.54)
- **强制置顶**：通过 DOM 强制置顶“全选”容器，确保其始终位于列表最上方。
- **垂直流向**：修正了宿主容器的 Flex 布局方向，解决了元素意外并排的问题。

### 3. 主题与品牌回归 (v4.52/4.51)
- **亮暗适配**：基于 `body.dark-theme` 的全自动色彩变量切换。
- **经典图标**：回归 Google Red (#EA4335) 设计，并优化为 PNG 格式以提升浏览器兼容性。
